## 2024-10-27 - [Framer Motion Type Narrowing]
**Learning:** When using Framer Motion with TypeScript, defining transition properties outside the component can lead to type widening issues (e.g., `type: "spring"` becomes `string`). This causes build failures in Vite/TypeScript projects.
**Action:** Always use `as const` when extracting Framer Motion variants or transitions outside the component body (e.g., `type: "spring" as const`) to ensure TypeScript correctly infers literal types and passes the strict `--noEmit` checks.
