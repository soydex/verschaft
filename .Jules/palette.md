## 2024-05-15 - FAQ Accordion Accessibility
**Learning:** Accordion components using `focus:outline-none` hide the default focus outline, making it completely invisible to keyboard users without a fallback. Additionally, mapping `id`, `aria-controls`, and `aria-labelledby` properly is crucial for screen readers to navigate complex UI elements correctly.
**Action:** Always provide `focus-visible:ring-*` as a fallback for custom components that hide default focus indicators, and ensure all accordion headers and contents are correctly cross-referenced via ARIA attributes.
## 2024-05-24 - Waitlist Form Accessibility Improvements
**Learning:** Found an accessibility anti-pattern where an input relied solely on its `placeholder` attribute for context, missing an explicit label. This is a common issue for single-input forms like waitlists or newsletter signups, which can be disorienting for screen reader users. Additionally, lacking a `disabled` state on successful form submission can lead to multiple unintended submissions and confusing user feedback.
**Action:** Always add an `aria-label` to inputs when a visible label isn't present, especially when they only have placeholders. Ensure form submit buttons implement a `disabled` state (along with visual cues like `disabled:opacity-75 disabled:cursor-not-allowed`) during and after processing, and remove interactive hover/tap animations while disabled to provide clear and consistent feedback to users.
